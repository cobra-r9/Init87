# What this really is 

This is about a template structure for the standard C projects. 
If I am gonna git push it, then it should contain the following by default. 

- .gitignore : the directories not to track via git. 
- .git : the git folder.
- Makefile : the build instructions for the program. 
- include : directory for all the c header files. 
- src : directory for all the c source files. 
- obj : directory for all the c object files. 
- readme.md : explaining the project stuff etc. 
- LICENSE : I would prefer MIT in this. case.
